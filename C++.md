## 内存分区模型

C++在程序运行时，操作系统把物理内存或虚拟地址空间划分为不同的区域进行管理，通常划分为以下几个主要分区：
1. 代码区：存放程序代码转化成的二进制机器指令，通常是只读的(频繁访问的程序，内存中只分配一块代码)且共享的(防止程序修改了它的指令)。 如函数定义、变量声明、类定义等。  
   
2. 数据区：存放已经初始化的数据，如全局变量、静态变量、常量(字符串常量、const修饰的全局变量)。也存放未初始化的数据，如未初始化的全局变量等，该区域的数据在程序结束后由操作系统释放。  

3. 堆区： 由程序员手动申请和释放的内存，从低地址到高地址增长，若最终内存未释放，则程序运行结束，该内存空间就会回收。

4. 栈区：存放函数的局部变量、函数参数、返回地址等，由系统自动分配和释放，从高地址到低地址增长。

### 内存分区变化过程

1. 程序运行前，编译代码：

程序编译后生成.exe可执行程序，此时分为代码区、数据区。  

2. 程序运行时，加载与执行：  

栈区动态分配：程序从 main 函数开始执行，系统自动为 main 函数分配 “栈帧”，存放其局部变量、函数参数等；当调用其他函数时，新的栈帧会被 “压入” 栈顶，函数执行完毕后，栈帧自动 “弹出” 并释放内存，栈区从高地址向低地址增长。

堆区手动管理：程序运行过程中，程序员可通过new（C++）或malloc（C）手动在堆区申请内存，堆区从低地址向高地址增长；申请的内存需通过delete或free手动释放，若未释放，程序结束后由操作系统统一回收。
#### C++中的new和delete(申请分配和释放堆内存)：     
new 数据类型(初始化数据); ----> 返回对应数据类型指针     
delete 指针名;  ----> 释放内存  
new 数据类型[数组大小];     ----> 返回对应数据类型的指针  
delete[] 指针名;  ----> 释放数组内存  

## 引用
引用是在给一个变量起别名,别名也指向原变量的内存,使原变量和别名共享同一块内存(显然，函数局部变量的引用，会随函数结束也一起无意义)，绝大部分情况下二者一样无差别。 值得注意的是，引用的编译器底层实现上，引用实际上是个指针常量，函数传参等情况时，实际上是在传递地址。

` 数据类型 &变量名 = 变量;`

引用的几个特性：  
1. 引用必须初始化，不能为空。  
2. 引用不能改变，引用对象不能改变。 

引用的主要应用是函数：  
1. 函数形参作引用时，形参的改变能改变实参。  
2. 函数返回引用(注意不能是局部变量的引用)时，常常通过返回引用的方式，返回函数内的局部静态变量(不随函数结束而释放，下次调用函数时，静态变量的值还能使用)。这时候也能把函数作为左值，能够修改所返回的引用变量值。
3. 常量引用：即一个只读的引用，不能改变引用对象。  
` const 数据类型 &变量名 = 变量;`  
a. 这样的引用能直接给数据起别名，实际上是编译器自动补全了变量的定义和初始化。  
b. 这样的引用常常作为函数形参使用，能够避免函数形参改变实参，而又能避免直接传入实参时的拷贝，代码速度更快。

# 函数高级
## 函数默认参数和占位参数
声明或者实现函数的时候，可以给函数的参数设置默认值。

当调用函数时，如果没有为该参数赋值，则使用默认值；  
所定义的函数参数的默认值，必须位于参数列表的最后；  
声明或实现函数中设定默认参数，只能任选其一。

`int add(int a, int b = 0) `

函数的占位参数就是函数声明或实现时的无名形参(可设定默认参数)：

`int add(int a, int ) `  
`int add(int a, int 10)`  

这样的函数调用时仍然需要给占位参数传参。  
函数的声明或实现中，占位参数可以随意搭配，无影响。 

## 函数重载

同一个作用域下的同名函数，能根据形参类型做区分，调用时的传入实参能够对应形参类型来选择具体的函数。

#### 函数重载的三个条件：  
1. 同名函数在同一个作用域下
2. 同名函数彼此之间的形参定义，有个数、数据类型或顺序上的区别。

##  B.对象特性
#### a.构造函数和析构函数
构造函数在对象创建时强制自动调用给对象属性初始化。  
析构函数在对象销毁前(生命周期结束，delete等)，强制自动调用执行清理工作，如释放对象申请的堆内存、关闭文件、释放锁等。
``` 
构造函数：      析构函数：
类名(){}        ~类名(){}
```
构造函数能有形参，可以发生重载(无参构造的使用不能加小括号，因为会跟函数声明冲突)；  
析构函数不能定义形参，无重载。

有参构造：能用传入的数据，在定义对象时需要传参  
`tree branch("apple",23); -----> tree(string fruit,int num){}`  
上述是经典的括号法定义，还有显示法：  
`tree first = tree("apple",23);`->语法逻辑上，右边创建了一个匿名对象(没有对象名)，左边复制这个对象，该行结束后立马回收这个匿名对象，这样的逻辑对一般的直接创建匿名对象是对的(而真实的编译器运行逻辑是直接利用右边数据初始化对象，对象的名字来来自于左边，根本没有所谓的匿名对象产生)  
还有隐式转换法：
`people first = 10;`->就是个显示法的简写，但只能对含一个形参的构造函数才能这样写  

#### 注意：不能对匿名对象使用拷贝构造！ 因为C++ 有一个最令人头痛的解析规则：只要代码能被解析成函数声明，就一定按函数声明来解析。tree(branch);由于C++的变量外加括号不影响、函数声明时也能把函数名加括号、无参函数的声明中参数列表小括号能省略，所以在编译器看来这样的指令就是 tree branch();在声明变量和声明函数中，编译器会最终选择声明函数。


#### b.拷贝构造
是一种特殊的有参构造，构造函数的形参是一个同类对象，形参有特定的写法：
`类名(const 类名 &另一个对象) {}`  
##### 拷贝函数的调用时机：
1. 用一个已经存在的对象去初始化一个新对象(构造函数一般用法)
2. 等号初始化(隐式转换法)`person p1 = p2;`
3. 作为实参传入函数形参(形参在初始化)
4. 作为函数返回值传入局部对象(局部对象在初始化)

##### 由此可见拷贝函数的形参为什么一定是要用引用：
函数形参本质上还是在从实参上进行值传递复制，过程中会有初始化，形参会调用拷贝函数。普通的形参`类名 形参名`在传入实参赋值给形参时，形参本身又调用了一层相同的内层拷贝函数，以此类推，形成拷贝函数的无限循环。而用&引用则本质上是个指针常量，直接传入了实参地址，没有所谓的形参复制，也就没有内层拷贝函数的调用。而用常量指针常量，是为了防止拷贝函数中的意外操作对实参值进行修改。

#### c.构造、拷贝和析构函数的调用规则
不手动书写这三种函数的情况下，编译器自动补全对应的默认函数：
``` 
构造：   默认无参构造，空函数体
拷贝：   默认浅拷贝(逐个复制成员属性值)
析构：   默认空函数体
```
注意的是：  
实际操作中，手动书写了有参构造，一般还需要多写无参构造，达到工程实践的标准化
#### d.深拷贝与浅拷贝
1. 浅拷贝  

浅拷贝是编译器默认的拷贝构造，它会默认将传入对象的成员属性的各项值赋值给新对象。

###### 这就出现了一个问题：如果成员属性是指针等类型，已存在的对象，它的成员属性是在堆区手动开辟的内存然后赋值，指针本身的存储是分配内存的地址。如果此时再默认浅拷贝一个新对象，新对象的成员属性是在把内存的地址复制，导致两者的地址都是指示在同一块堆内存上。一旦修改一个对象的成员属性，另一个也会修改；一个成员属性释放，另一个成员属性直接非法变空(NULL)。

2. 深拷贝

深拷贝要求编译者必须要手动书写构造函数，构造函数里明确指示新对象的指针类成员属性要先手动分配堆内存，然后再赋值初始化。

一般的，为了节省程序运行时的内存，需要在析构函数里delete所分配的堆内存。
#### e.初始化列表
初始化列表也是构造函数的一部分，它是在成员属性变量创建的同时(这个时间在构造函数进入函数体之前)给其赋值(而构造函数内的初始化是先创建成员属性变量再进入函数体等号赋值)，效率相对更高。
```
类名(形参列表) : 成员1(值), 成员2(值), 成员3(值)
{
    // 构造体
}
```
##### 初始化列表主要的作用是针对于成员属性是常量、引用或其他类对象(其类的定义中没有无参构造)的情况的，他们必须要在变量创建的同时指定储存值。
另外需要注意的是，初始化列表的书写顺序必须跟类内成员属性定义的顺序相同(因为成员属性变量的创建跟其先后定义顺序相同，而初始化列表的初始化方法要求必须在变量创建的同时存入变量)，否则部分成员属性没有赋到有效值。

#### d.其他类对象作为类内成员属性

B类的定义属性成员中有A类对象,这个对象称为对象成员

创建外部类对象：成员对象先构造，外部类对象再构造  
销毁外部类对象：外部类对象先析构，成员对象再析构

#### f.静态成员
static修饰下的成员属性和成员行为，统称静态成员  
静态成员分两种：静态成员变量 + 静态成员函数

静态成员变量：  
``` 
类内：static int a;         static修饰成员属性
类外：int 类名:: a = 100;    类外初始化静态变量
```
1. 程序开始的编译阶段就创建并分配内存(全局区)，程序结束才销毁，全局只创建一个
2. 属于类内共享变量，所有对象都能共享操作静态变量的数据
3. 除了对象.访问，`类名::静态变量名`也能访问  

静态成员函数：  
```
static void exmple(){} 
```
1. 静态成员函数是属于类的函数，对象调用静态函数，函数无法通过this指针访问调用者对象的成员属性。
2. 函数内部能够直接调用静态成员变量
3. 除了对象.访问，`类名::静态函数名()`也能访问 
4. 若是类内声明类外实现，实现的时候不用写static修饰

## C.对象模型
类对象的内存分配如下：
```
非静态成员变量  栈/堆内存   属于对象内存 每个对象独自一份
静态成员变量    全局数据区  属于类       共享唯一
成员函数        代码段     属于类       共享唯一
```
对一个对象来说，它的内存大小就是非静态成员变量的内存对齐结果。  
空类定义的空对象，也分配了1字节内存；

#### this指针
this 指针是非静态成员函数的隐藏形参(设定在第一个,是个指针常量)，由编译器自动添加，指向当前调用该函数的对象的内存***地址***。

`tree apple() -->  tree apple(tree* const this)`

仅在成员函数调用期间存在（栈上临时变量），函数执行完自动销毁，不是对象的成员，不占对象内存。  
`this->apple`    ->   表示正在调用该函数对象的apple成员属性

主要的用途：
1. 解决形参名称和成员属性名称冲突的问题
2. 函数返回值作*this，函数返回值类型设置为：类名& 也就是引用，能返回对象本体，可构建链式调用  
(若设置返回值类型为 类名 ，返回"值"的数据类型，则函数会先处理创建一个临时副本再返回，一般到达;后销毁，而且不允许用非const引用把临时变量调入其他函数，这个副本跟原来的对象是两个东西)

##### 空指针访问成员函数

类定义的空指针，能访问类的成员函数，前提是不涉及成员函数对成员属性操作或者是静态成员函数(因为this隐藏形参的作用，空指针没有成员属性)

##### const修饰成员函数和对象 
const修饰成员函数，即常函数，特性是不能在该函数中修改属于对象的成员，定义是如下的方式:  
`返回值类型 函数名(形参列表) const {} `  
那么该函数的隐藏形参this如下定义:  
` const 类名* const this`  
也就是说this不能改动指向也不能改动指向地址的储存值。  
在定义类的属性时，用mutable修饰的成员属性则能够在常函数中改动其指向地址的存储值。  
` mutable int a;`  

const修饰对象，即常对象，特性是不能通过该对象修改属于对象的成员:
`const tree apple; `  
不能修改对象的成员属性(除了mutable修饰的)  
只能通过常对象调用常函数  

那么根据两者的特性，一般有如下规律：
1. const成员函数能读对象属性，只能修改mutable属性和静态属性，只能调用其他的const成员函数
2. const修饰对象能读对象属性，只能修改mutable属性和静态属性，只能调用const成员函数

## D.友元
友元(friend关键字修饰)可以让指定的外部函数/类，直接访问另一个类的私有(private\peotected)成员，而无需通过 public 接口。
1. 全局函数作友元  

一个类的私有成员对一个全局函数内部调用开放，在类的定义里写注明：  
` friend 函数声明格式;`

2. 类作友元(友元类)

让一整个类成为友元，该类的所有成员函数都能直接访问原类的私有成员。让类A成为类B的友元，则就在类B的定义中写注明：    
` friend class A;`

3. 友元成员函数  

让一个类的某个成员函数成为一个类的友元，这个成员函数内能直接访问原类的私有成员。  
` friend 函数声明格式(名称带上类名::);`
但是总体上有固定的书写顺序：
  - 先写原类的声明：class 类名；
  - 再写友元类的定义，友元成员函数只写一个声明；
  - 再写原类的定义，加上friend声明；
  - 最后在类外定义实现友元成员函数

## E.运算符重载
C++ 运算符重载相当于给运算符起了外号，让它能用来操作自定义对象，本质上是个自定义函数。  

在定义函数时，用`operator重载符号`,作为函数名在调用时能直接简写为运算符的运算写法;  
也能定义同符号的不同函数，可以发生重载;友元可以应用于运算符重载函数;  
一般的，运算符的运算不改变输入变量(右边对象)的值，设定的函数形参一般是常量引用，如果也不改变右边对象的值，设定的函数一般为常函数。  

#### +重载(同-)
1. 成员函数写法(适合对象和对象的操作)
2. 全局函数写法(适合对象和其他数据的操作)
#### <<左移运算符
只能用全局函数书写,cout的数据类型是ostream,下有固定写法：
```
ostream& operator<<(ostream& cout, const tree& t) {
    cout << t.a << " " << t.b;
    return cout;  //满足链式调用
}
```
#### ++递增运算符重载(同--)
一般用成员函数书写，下有固定写法：
```
// 前置++：无参数，返回引用
类名& operator++() {
    // 自增逻辑
    return *this; // 返回自增后的对象本身
}

```
##### 后置++，函数需要占位参数以跟前置区分
```
// 后置++
类名 operator++(int) {
    类名 新临时对象 = *this;
    // 自增逻辑(对当前对象)
    return 新临时对象; // 返回旧的临时对象
}
```
#### =赋值运算符重载
一般用成员函数书写。  
实际上，类的默认函数里就有赋值运算符的重载函数，默认是：
tree& operator=(const tree&)，内部是一个浅拷贝，只处理了成员属性的拷贝，而没有处理之前考虑的深拷贝问题。  

先判断自赋值(返回自己)->再释放原指针空间->创建新空间并用指针指示->新空间赋值

``` 
类名& opreator=(const tree& 对象名){
    //处理辅助
    return *this;
}
```
#### 关系运算符重载
成员函数和全局函数都可以。类似于以下写法：  
```
bool operator==(const 类名& other) const {
    // 比较成员变量，返回 bool
    return (a == other.a && b == other.b);
}
```
#### 函数调用运算符重载()
()重载后的使用就像一个函数，常被称为仿函数。
只能用成员函数实现。
```
返回值类型 operator()(形参列表) {
    // 函数体
}
```
##### 补充匿名
匿名对象：没有名字，临时，一行就死。`类名();`  
函数对象：重载了operator()的对象,能当函数用。`对象(参数); `  
匿名函数对象：既是匿名对象，又是函数对象。`类名(参数);`
