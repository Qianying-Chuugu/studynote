4. 测试文件.cpp中调用函数：

### 指针：储存内存地址的变量
#### const修饰指针
1. 常量指针：  
   const int *p;  
   常量指针的本身存储的地址值可以改，但地址值所对应的数据不能通过这个指针改。
2. 指针常量：  
   int *const p;
   指针常量的本身存储的地址值不能改，但地址值所对应的数据可以通过这个指针改。
3. 指针常量指针：  
   const int *const p;
   指针本身存储的地址值和地址值所对应数据不能通过这个指针改。
   
### 结构体：储存多个数据类型的自定义数据类型
兼容C，但是有C++中class的几乎全部功能，后面介绍，注意struct跟class的两个区别即可：
1. 不规定访问权限的情况下，class所有成员都是private的，struct所有成员都是public的。
2. 不规定继承方式的情况下，class所有成员都是private继承(访问权限都是private)，struct所有成员都是public继承(访问权限对齐父类的设定)

## 内存分区模型

C++在程序运行时，操作系统把物理内存或虚拟地址空间划分为不同的区域进行管理，通常划分为以下几个主要分区：
1. 代码区：存放程序代码转化成的二进制机器指令，通常是只读的(频繁访问的程序，内存中只分配一块代码)且共享的(防止程序修改了它的指令)。 如函数定义、变量声明、类定义等。  
   
2. 数据区：存放已经初始化的数据，如全局变量、静态变量、常量(字符串常量、const修饰的全局变量)。也存放未初始化的数据，如未初始化的全局变量等，该区域的数据在程序结束后由操作系统释放。  

3. 堆区： 由程序员手动申请和释放的内存，从低地址到高地址增长，若最终内存未释放，则程序运行结束，该内存空间就会回收。

4. 栈区：存放函数的局部变量、函数参数、返回地址等，由系统自动分配和释放，从高地址到低地址增长。

### 内存分区变化过程

1. 程序运行前，编译代码：

程序编译后生成.exe可执行程序，此时分为代码区、数据区。  

2. 程序运行时，加载与执行：  

栈区动态分配：程序从 main 函数开始执行，系统自动为 main 函数分配 “栈帧”，存放其局部变量、函数参数等；当调用其他函数时，新的栈帧会被 “压入” 栈顶，函数执行完毕后，栈帧自动 “弹出” 并释放内存，栈区从高地址向低地址增长。

堆区手动管理：程序运行过程中，程序员可通过new（C++）或malloc（C）手动在堆区申请内存，堆区从低地址向高地址增长；申请的内存需通过delete或free手动释放，若未释放，程序结束后由操作系统统一回收。
#### C++中的new和delete(申请分配和释放堆内存)：     
new 数据类型(初始化数据); ----> 返回对应数据类型指针     
delete 指针名;  ----> 释放内存  
new 数据类型[数组大小];     ----> 返回对应数据类型的指针  
delete[] 指针名;  ----> 释放数组内存  

## 引用
引用是在给一个变量起别名,别名也指向原变量的内存,使原变量和别名共享同一块内存(显然，函数局部变量的引用，会随函数结束也一起无意义)，绝大部分情况下二者一样无差别。 值得注意的是，引用的编译器底层实现上，引用实际上是个指针常量，函数传参等情况时，实际上是在传递地址。

` 数据类型 &变量名 = 变量;`

引用的几个特性：  
1. 引用必须初始化，不能为空。  
2. 引用不能改变，引用对象不能改变。 

引用的主要应用是函数：  
1. 函数形参作引用时，形参的改变能改变实参。  
2. 函数返回引用(注意不能是局部变量的引用)时，常常通过返回引用的方式，返回函数内的局部静态变量(不随函数结束而释放，下次调用函数时，静态变量的值还能使用)。这时候也能把函数作为左值，能够修改所返回的引用变量值。
3. 常量引用：即一个只读的引用，不能改变引用对象。  
` const 数据类型 &变量名 = 变量;`  
a. 这样的引用能直接给数据起别名，实际上是编译器自动补全了变量的定义和初始化。  
b. 这样的引用常常作为函数形参使用，能够避免函数形参改变实参，而又能避免直接传入实参时的拷贝，代码速度更快。

# 函数高级
## 函数默认参数和占位参数
声明或者实现函数的时候，可以给函数的参数设置默认值。

当调用函数时，如果没有为该参数赋值，则使用默认值；  
所定义的函数参数的默认值，必须位于参数列表的最后；  
声明或实现函数中设定默认参数，只能任选其一。

`int add(int a, int b = 0) `

函数的占位参数就是函数声明或实现时的无名形参(可设定默认参数)：

`int add(int a, int ) `  
`int add(int a, int 10)`  

这样的函数调用时仍然需要给占位参数传参。  
函数的声明或实现中，占位参数可以随意搭配，无影响。 

## 函数重载

同一个作用域下的同名函数，能根据形参类型做区分，调用时的传入实参能够对应形参类型来选择具体的函数。

#### 函数重载的三个条件：  
1. 同名函数在同一个作用域下
2. 同名函数彼此之间的形参定义，有个数、数据类型或顺序上的区别。

# 面向对象
一个类 = 成员变量 + 成员函数 + 访问权限
``` 
class 类名称{
    权限：
        定义属性；//成员属性
    权限：
        成员函数；//成员方法
};
```
类外的对象能使用相应权限的属性或函数，通过点.访问。 
## A.封装
#### 三种权限：  
1. public 公开权限，类内、类外都能访问
2. private 私有权限，只有类内自己能访问
3. protected 保护权限，类内 + 子类能访问(继承)

把属性（变量）藏起来，不让外面随便改，只留安全的接口（成员函数）给外面用，这就叫封装。
```
private成员---函数设定接口---函数检测数据有效性---函数修改对象属性
```

#### 类的分文件编写注意点：  
1. .h文件写入类的定义，把成员函数写成声明即可，其他跟普通声明一样
2. 源文件只写成员函数的具体实现，需要在函数名字前加上class作用域，具体为：
` int 类名：：成员函数名(){}`
